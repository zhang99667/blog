# MarkZ Public Notes Contract

本文档是公开笔记站的跨层契约。内容同步、Quartz 页面、图谱运行时、边缘路由和质量门禁都必须遵守同一组规则，不能分别定义一套“看起来能用”的行为。

## 1. 产品与内容边界

- 正式入口固定为 `https://note.markz.fun/`；`https://markz.fun/notes/` 只负责跳转，不承载第二份可索引内容。
- 笔记是保留网络结构和研究过程的工作台，博客是整理后的文章。笔记页可以有 Explorer、关系图谱、目录和回链，博客文章不复制这些组件。
- Obsidian Vault 是内容权威源。Markdown 只有 `publish: true` 才进入公开笔记；精确的 `type: post` 才额外进入博客。
- `content/notes/`、`public-notes/` 和博客中的笔记链接都是生成结果，不作为手工修复入口。

## 2. 内容格式契约

公开 Note 使用以下最小 front matter：

```yaml
---
publish: true
type: note
topic: AI
---
```

- `publish` 决定是否进入公开笔记站，只接受布尔值 `true` 或 `false`。
- `type` 决定内容形态；普通笔记固定为 `note`，需要同时进入博客的成稿使用 `post`。
- `topic` 是稳定主题，例如 `AI`、`Android`、`Network`，不能使用一次性任务名。
- 私有或未完成笔记使用 `publish: false`，但仍保留 `type: note` 与 `topic`，避免公开状态变化时再猜测内容归属。
- 存量旧笔记不做无关批量改写；新增笔记和本次修改到的旧笔记必须补齐最小 front matter。源 Vault 的项目级 Hook 负责检查本次变更范围。

## 3. 路由契约

- 每篇笔记只有一个 canonical 路由：对目标 Markdown 相对路径执行 Quartz `slugifyFilePath`，去掉扩展名、规范化大小写和空格，并保留可读的中文字符。
- 同步器生成的首页、图谱、搜索索引、博客正文和“原始笔记”链接必须消费同一个 canonical 路由，不得重新拼接原始标题 URL。
- HTML 使用无扩展名 clean URL；静态构建产物使用对应的 `<slug>.html`。构建质量必须能把所有 `note.markz.fun` 链接映射回真实的 `public-notes` 文件。
- 不存在的路径返回带 `noindex` 的笔记 404 页面和 HTTP 404，不能返回 HTTP 200 的笔记首页。历史非规范路径只有在维护显式 301 映射时才能继续使用。
- 404 与任意深层页面的 CSS、JavaScript、字体和品牌资产都必须从正确站点根路径加载；未命中页面不能退化成浏览器默认样式。

## 4. 页面与滚动契约

- `1440x900` 宽屏使用左侧 Explorer、中心正文和右侧阅读辅助区；中心正文保持稳定阅读宽度。`320x800` 与 `390x844` 使用单列阅读流，导航、搜索、标题和正文不得重叠或横向溢出。
- 左侧 Explorer 可以独立滚动。宽屏右栏跟随页面主滚动，关系图谱、目录和回链不得再形成第二条纵向滚动链。
- 本地关系图谱在宽屏保持 `10rem` 紧凑高度，画布的 CSS 宽高必须与 `.graph-container` 完全一致，力导向中心就是可见容器中心。全局图谱尺寸不受此限制。
- 图谱、目录和互动栏必须保留至少一个标准间距；目录末项能通过页面主滚动完整进入视口。
- 浅色与深色主题共享相同几何关系。主题变化只能改变语义颜色，不能改变路由、组件数量、画布尺寸或阅读顺序。

## 5. 导航与运行时契约

- Explorer、正文链接、搜索结果和图谱节点都导航到 canonical slug。路径进入运行时前必须解码为与内容索引一致的 Unicode slug。
- SPA 导航必须销毁旧图谱实例并只挂载一个新画布；主题切换可以重绘，但不能累积 Canvas、远程运行时或事件监听器。
- D3、Pixi、Mermaid 和页面脚本只从本站固定版本资源加载。CSP 违规、外部 CDN 回退或脚本失败不能影响静态正文阅读。
- 页面初次打开保持在顶部；只有显式锚点可以改变初始滚动位置。

## 6. 验收契约

- 同步和路由：`npm run sync`、`npm test`、`npm run build`、`npm run quality:build`。
- 页面体验：`npm run quality:web` 覆盖笔记首页与正文、`320x800`、`390x844`、`1440x900`、浅色和深色。
- 浏览器门禁至少检查 canonical 页面不是首页回退、主布局样式已生效、图谱画布与容器同宽同高、没有横向溢出和 CSP 违规。
- 上线后：`npm run smoke:production` 同时验证 canonical 笔记返回 200、未知笔记返回带 `data-slug="404"` 的 404、静态资源和安全响应头正常。

任何一层需要偏离本契约时，先更新本文件和 `docs/AI-DECISIONS.md`，再修改实现与自动检查。不能只在生成 HTML、Nginx 或截图阈值中做局部例外。
